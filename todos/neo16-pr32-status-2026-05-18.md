## What landed in this PR

1. **Reconciliation matching pipeline** (`convex/setReconciliation.ts`)
   - Strip SL Base prefix and drop the Base anchor row before auto-matching.
   - `PLURALIZABLE_WORDS` set + `singularize()` plugged into `normalizeForMatch`. Currently just `["prizm"]`; extend by adding singulars.
   - New bag-of-words pass between exact-normalized and fuzzy. Catches `Prizms Red` ↔ `Red Prizm` via sorted-token multiset compare.
2. **Reconciliation modal UX** (`components/SetSelector/ReconciliationModal.tsx`)
   - BSC column gets a matching contains-search filter for column parity with SL.
   - SL typed query decoupled from the prefix toggle — it is now a secondary contains-search inside the prefix-filtered list (or full list when `Show all` is on).
   - Shared `FilterInput` with inline `×` clear button.
   - New optional `levelLabel` prop so callers override the level-derived heading.
3. **Variant-type label propagation**
   - `VariantSelector` accepts a `title` prop.
   - `components/modules/SetSelector.tsx` stabilizes the variantType `value` across Convex refetches and feeds the pluralized form (`Inserts`, `Parallels`) into the column header, Sync button, and `addButtonText`.
   - `VariantForm` heading + loading copy use the same pluralized label, and pass it as `levelLabel` to `ReconciliationModal`.
4. **Race fix in `VariantForm.tsx`** — initial `doSync` `useEffect` now gates on `baseVariant !== undefined` so `baseSlPrefix` actually reaches `fetchRawOptions` on the first call.
5. **Maestro flow text updates** — 19 yaml flows updated for the new `Inserts` / `Parallels` / `Reconcile Inserts` / `Reconcile Parallels` / `Add custom Inserts` / etc. labels. Insert-branch flows use the Insert variant of each string; Parallel-branch flows use the Parallel variant. `wip` flows use a regex alternation pattern so they keep working if un-wip'd.
